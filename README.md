# Zenzele Smart Market

## “Be Self-Reliant”

**Zenzele Smart Market** is an open-source, web-based entrepreneur marketplace built on the secure and scalable **Cardano blockchain**. The word **Zenzele** means **“be self-reliant”**, and the project is designed to help unemployed people, small business owners, freelancers, creators, students, artisans, and everyday hustlers from **any country** promote themselves, find support, receive funding, request training, and grow real economic activity. 

[Google Hussle Academy](https://www.itweb.co.za/article/google-opens-hustle-academy-2026-registration/kLgB17ezVOaM59N4)

The platform is not only for Africa. It must work for entrepreneurs in South Africa, Zimbabwe, Nigeria, Kenya, India, Brazil, the United States, the United Kingdom, Europe, Asia, and any other country where people need a simple way to promote their work and build income. Because of this, Zenzele Smart Market must support multiple languages. The first version can begin with English, isiZulu, isiXhosa, Sesotho, Setswana, Afrikaans, and Swahili, but the design must allow more languages to be added later without rebuilding the whole system.

Zenzele Smart Market will allow a user to sign up, log in, create a profile, describe their business or hustle, record an audio pitch, add soft background music, create an entrepreneur NFT, request donations, request training, receive followers, and share their profile on WhatsApp, TikTok, Facebook, Instagram, Telegram, and X. The goal is to make the app simple enough for a normal person with little technical knowledge to use, while still giving developers enough technical depth to build secure, high-quality Cardano features.

The project must be fast, lightweight, and friendly to users with limited data. It should avoid unnecessary heavy frameworks and should work well on phones, tablets, laptops, and desktop browsers. Because it is web-based, it becomes cross-platform without needing separate Android, iOS, Windows, and Linux apps.

## Technology Requirements

The project must use simple and accessible technologies:

The frontend must use **HTML, CSS, and JavaScript**.

The backend must use **PHP and MySQL**.

The Cardano smart contracts must use **Haskell / Plutus**.

The Cardano off-chain transaction logic must use **Lucid**.

Testing must include strong unit-quality tests, smart contract tests, and user-flow tests. Developers must show that important features work correctly, including login, profile creation, NFT metadata creation, donation recording, following, likes, views, and training requests.

The source code must be fully shared on GitHub. No private keys, passwords, seed phrases, API secrets, or production credentials may be committed. Developers must fork the **zenzelesmartmarket** repository, make their changes, and submit pull requests.

## Main User Experience

A new user should be able to open the website and immediately understand what Zenzele Smart Market does. The homepage should explain that the platform helps people become self-reliant by promoting their skills, products, services, and business ideas.

A person who is unemployed should be able to join the platform and create a profile that says what they can do, what they want to learn, what support they need, and what kind of work they are looking for. A small business owner should be able to list their products or services, upload images, record an audio pitch, and ask for funding or training. A supporter should be able to browse entrepreneurs, listen to their stories, follow them, like them, share them, and donate through Cardano.

The profile page should feel like a professional mini-business page. It should show the person’s name, location, business category, skills, story, wallet address, social links, NFTs, donation requests, training requests, followers, following, views, likes, hearts, stars, and activity history.

The audio feature is important because many users may not write long descriptions well. They should be able to record a short voice note explaining who they are, what they do, what help they need, and how they plan to use support. The app should also allow them to add soft background music, but the music must not overpower the voice. The user should be able to preview the audio before saving it.

The NFT feature should allow entrepreneurs to mint a digital proof of their business, product, service, achievement, or campaign. The NFT metadata should include the title, description, image, audio story, category, owner wallet address, timestamp, and support goal. The NFT must not be a confusing crypto-only feature. It should be explained to ordinary users as a digital certificate of their hustle, business, story, or achievement.

The donation feature should allow supporters to help entrepreneurs directly. Donations should be transparent, and where possible, Cardano transaction references should be visible. In future versions, milestone-based donations and escrow can be added so that supporters can fund specific goals such as tools, training, stock, transport, school fees, equipment, or business expansion.

The training feature should allow users to request help in areas such as coding, AI, farming, digital marketing, business management, finance, fashion, music production, carpentry, graphic design, sales, bookkeeping, and entrepreneurship. Mentors should be able to view requests, accept them, and eventually issue NFT certificates for completed training.

## Global Language Support

Zenzele Smart Market must be designed for international use. The language system should not hard-code text directly into pages. Instead, user interface text should be stored in language files or database translation tables. This will allow contributors from different countries to add translations.

The app should support country, language, and currency preferences. A user in South Africa might prefer English or isiZulu. A user in Kenya might prefer English or Swahili. A user in Brazil might prefer Portuguese. A user in India might prefer Hindi, Tamil, or English. The system should be flexible enough to grow.

## AI Usage and Reproducibility

This is an AI-created developer contest, so every team must clearly document how they used AI. Developers must include the exact AI prompts they used, the type of LLM used, the model name where possible, and what the AI helped with.

For example, a developer must say whether AI was used to generate database schemas, improve UI text, create test cases, write smart contract drafts, translate interface text, review security issues, or improve documentation.

Every AI-generated section must be reproducible by others. That means contestants should include a file such as:

`AI_USAGE.md`

This file should contain the model used, the date used, the prompts, the outputs, what was accepted, what was rejected, and what was manually edited. This will help judges understand the developer’s real work, code quality, and responsible AI usage.
